基礎講座 第4章

ルーティング — OSPF・BGP・経路制御

ルータが「どの経路でパケットを送るか」を決める仕組みを学ぶ。ルーティングテーブルを読めるようになり、OSPFとBGPの動作原理と設計の勘所を、午後問題で問われるレベルまで引き上げる章だ。

学習目標:
  • ルーティングテーブルからロンゲストマッチで転送先を判定できる
  • 直結・スタティック・動的経路の優先順位(アドミニストレーティブディスタンス)を説明できる
  • OSPFのネイバー・DR/BDR・LSA・エリア設計の役割を説明し、コスト計算で最短経路を求められる
  • BGPのAS_PATH・LOCAL_PREF・MEDを使い分けて経路制御の方針を答えられる
  • 経路再配布で起きる問題(ループ・メトリック変換)を指摘できる

想定学習時間: 4〜5時間(例題・章末テスト込み)

4-1ルーティングテーブルの読み方

ルータの仕事はただ一つ、「届いたIPパケットの宛先IPアドレスを見て、次にどこへ渡すかを決めて送り出す」ことだ。この判断の根拠になる表がルーティングテーブル(経路表)。NW試験の午後問題では、ルーティングテーブルの抜粋がそのまま問題文に載り、「この宛先のパケットはどのインタフェースから出ていくか」を答えさせる。まずこの表を正確に読めることが全ての出発点になる。

ルーティングテーブルの1行(1エントリ)は、基本的に次の3点セットでできている。

  • 宛先ネットワーク — ネットワークアドレスとプレフィックス長(例: 10.2.5.0/24)。個々のホストではなく「ネットワーク単位」で持つのがポイント。だから全世界のホストを知らなくても転送できる。
  • ネクストホップ — そのネットワーク宛のパケットを次に渡すルータのIPアドレス。自分に直接つながるネットワーク(直結経路)ならネクストホップは不要で「directly connected」などと表示される。
  • 出力インタフェース — パケットを送り出す自分のインタフェース。

実際の表にはこのほか、経路をどう学んだか(直結/スタティック/OSPF/BGPなど)と、メトリック(その経路の「良さ」を表す値)が載る。ここで大事な感覚がひとつ。ルーティングはホップバイホップ、つまり各ルータがそれぞれ自分のテーブルだけを見て独立に判断する。送信元のルータが経路全体を決めるわけではない。だから「行きのパケットはルータAを通るが、帰りはルータBを通る」という非対称ルーティングも普通に起こり得る。往路と復路を別々に追いかける癖は、午後問題を解くうえで必須の作法だ。

        [PC] 10.1.1.10
          │
      ── 10.1.1.0/24 ──
          │
        [R1] ルーティングテーブル(抜粋)
          │    宛先            ネクストホップ  出力IF
          │    10.1.1.0/24     直結           ge-0
          │    10.2.5.0/24     10.255.0.6     ge-2
          │    0.0.0.0/0       203.0.113.1    ge-1

合格の視点ルーティングテーブルの各行は「宛先の範囲」と「次の一手」の対応表にすぎない。表が読めれば、OSPFもBGPも「この表を誰がどう作るか」の話でしかない。第3章のサブネット計算が怪しい人は先に戻っておこう。この章の計算はすべてサブネットの理解の上に載っている。

4-2ロンゲストマッチとデフォルトルート

宛先IPアドレスが複数の行にマッチしたら、ルータはどうするか。答えは決まっていて、プレフィックス長が最も長い(=範囲が最も狭い)行を採用する。これがロンゲストマッチ(最長一致)の原則だ。「より具体的な情報を優先する」と考えれば自然だろう。10.2.0.0/16と10.2.5.0/24の両方にマッチする宛先なら、/24の行が勝つ。

この原則の対極にあるのがデフォルトルート 0.0.0.0/0。プレフィックス長0、つまり「全てのIPアドレスにマッチするが、優先度は最弱」という行だ。どの行にもマッチしなかったパケットの最後の受け皿になる。企業ネットワークでは「社内の経路は個別に持ち、インターネット宛はまとめてデフォルトルートでファイアウォールへ」という設計が定番で、午後問題の構成図でも頻出する。逆に、デフォルトルートすら無い宛先のパケットは破棄され、ルータは送信元へICMPエラー(宛先到達不能)を返す。

ロンゲストマッチは経路制御の道具としても使われる。例えば全体を10.2.0.0/16で広めておき、特定の10.2.5.0/24だけ別経路に向ける、という「例外は細かい経路で上書きする」テクニックだ。経路集約(4-7)と組み合わせると、テーブルを小さく保ちながら例外だけ制御できる。

試験ではこう出る午前IIでは「ルーティングテーブルと宛先IPアドレスを与えて転送先を選ばせる」問題がほぼ定番。/16と/24と/25が混在した表で、ロンゲストマッチを正確に適用できるかが問われる。宛先アドレスとサブネットマスクのAND演算を暗算でなく手を動かして確認する習慣をつけると事故が減る。

4-3経路の優先順位 — アドミニストレーティブディスタンス

ロンゲストマッチは「テーブルの中でどの行を使うか」の話だった。その一歩手前に、もうひとつ選抜がある。同じ宛先ネットワーク(同じプレフィックス長)の経路情報を、複数の情報源から同時に学んだら、どれをテーブルに載せるかという問題だ。例えば10.2.5.0/24への経路を、管理者が手で設定したスタティックルートとしても、OSPFからも学んでいる、という状況は普通にある。

この優劣を決める値がアドミニストレーティブディスタンス(AD)。「その情報源をどれだけ信用するか」を数値化したもので、小さいほど信用度が高い。序列の考え方はシンプルだ。

  • 直結経路が最強 — 自分のインタフェースに直接つながっているのだから疑いようがない。
  • 次にスタティックルート — 管理者が意図をもって手で書いた経路。
  • その次に動的ルーティングプロトコル — プロトコルが自動計算した経路。プロトコル間にも序列がある。

表4-1: AD値の代表例。具体的な数値は実装(ベンダ)によって異なるが、「直結>スタティック>動的」という序列の考え方は共通だ。数値そのものより序列で覚える。

整理すると、ルータの経路選択は2段階で動く。(1) 同じ宛先プレフィックスの候補が複数あれば、ADが最小の情報源の経路をテーブルに載せる(同一プロトコル内ならメトリックが最小のもの)。(2) パケット転送時には、テーブルに載った行の中からロンゲストマッチで1行を選ぶ。ADは「テーブルに載せる経路の選抜」、ロンゲストマッチは「転送時の行の選択」——この役割の違いを混同しないこと。/16のOSPF経路と/24のスタティック経路は宛先が違うので競合せず、両方テーブルに載る。/24宛のパケットには(ADに関係なく)/24の行が使われる。

フローティングスタティック — AD値を逆手に取った予備経路

ADの仕組みを応用した定番の冗長化テクニックがフローティングスタティックルートだ。スタティックルートにわざと大きなAD値(例: 250)を設定しておく。すると通常時はOSPF(AD 110)の経路が勝ってテーブルに載り、スタティックルートは「浮いた」状態で待機する。主回線が切れてOSPF経路が消えた瞬間、待機していたスタティックルートがテーブルに昇格し、予備回線経由の通信が自動的に始まる。

        │                                          │

4-4ダイナミックルーティングの分類 — ディスタンスベクタとリンクステート

スタティックルートは単純で確実だが、ネットワークが大きくなると破綻する。ルータが50台あれば、宛先ネットワーク×ルータ台数分の設定を人手で管理し、障害や増設のたびに全台を直して回ることになる。設定ミスも起きるし、回線断への自動対応もできない(フローティングスタティックは「あらかじめ書いた予備」までだ)。そこで、ルータどうしが経路情報を自動交換して経路を計算するのがダイナミックルーティングプロトコルである。

ダイナミックルーティングプロトコルは2つの軸で分類できる。1つめの軸は使う場所。AS(自律システム——ひとつの管理ポリシーで運用されるネットワークの単位。4-8で詳述)の内側で使うものをIGP(Interior Gateway Protocol: RIP、OSPFなど)、AS間で使うものをEGP(Exterior Gateway Protocol: 現在は事実上BGPのみ)と呼ぶ。2つめの軸は経路計算のアルゴリズムで、こちらが本節の主役だ。

  • ディスタンスベクタ型(RIPが代表) — 各ルータは「宛先ネットワークまでの距離(ホップ数)と方向(ネクストホップ)」だけを隣のルータに定期的に伝える。いわば「隣の噂話」の伝言ゲームで、各ルータはネットワーク全体の姿を知らない。仕組みが単純な反面、伝言が一巡するまで収束が遅く、経路情報がループして距離が無限に増え続ける「カウントトゥインフィニティ」問題を抱える。RIPはホップ数16を到達不能とみなすことでこれを打ち切るが、裏を返せば15ホップまでの小規模網にしか使えない。また距離=ホップ数なので、低速な1ホップを高速な2ホップより「良い」と誤判定する。
  • リンクステート型(OSPFが代表) — 各ルータは「自分にどんなリンク(回線)がつながっているか」という一次情報をネットワーク全体に配布(フラッディング)する。全ルータが同じ「地図」(リンクステートデータベース)を持ち、各自がその地図の上で最短経路を計算する。伝言ゲームではなく地図の共有なので、収束が速く、ループも起きにくい。帯域幅ベースのコストで経路を選べる。その代わり、地図の維持と計算にCPU・メモリを使う。

4-5OSPF(1) — Hello・ネイバー・隣接関係とDR/BDR

ここからOSPF(Open Shortest Path First)を3節かけて掘り下げる。OSPFは企業ネットワークのIGPとして最も広く使われ、NW試験でも設計・障害解析の題材として最頻出クラスだ。まずはルータどうしが「知り合い、地図を交換する」までの流れから。

OSPFルータは各インタフェースからHelloパケットをマルチキャスト(224.0.0.5宛)で定期送信する(イーサネットでは既定10秒間隔)。Helloには自分のルータID(ルータを識別する32ビット値)やエリアID、Hello間隔などが入っており、同じセグメントで互いのHelloを受け取ったルータどうしはネイバー(隣人)として認識し合う。このときエリアID、Hello間隔・Dead間隔(Helloが途絶えてから死亡判定するまでの時間。既定はHelloの4倍の40秒)、認証設定、サブネットが一致しないとネイバーになれない。「OSPFが上がらない」障害の定番原因なので、この一致条件は覚えておく価値がある。

ネイバーになっただけでは経路情報は流れない。LSDB(地図)を同期し合う特別な関係——隣接(アジャセンシー)関係——まで進んだペアだけが経路情報を交換する。そしてここで問題がひとつ。イーサネットのようなマルチアクセスネットワーク(同一セグメントに3台以上のルータがつながれる網)では、n台のルータが全員と隣接を結ぶと n(n−1)/2 本の関係が必要になる。10台なら45本。同じ地図を45本のパイプで重複交換するのは明らかに無駄だ。

そこでOSPFは、セグメントごとにDR(代表ルータ)とBDR(バックアップ代表ルータ)を選出し、LSAの交換をDRに集約する。DR/BDR以外のルータ(DROTHER)はDRとBDRだけと隣接関係を結び、経路情報の更新はDR宛のマルチキャスト(224.0.0.6)に送る。DRがそれをセグメント全体に再配布する。これで交換の本数はほぼ台数に比例する程度まで減る。BDRはDRの死活を監視し、DR障害時に即座に昇格する。

   DR/BDRなし: 全員が全員と隣接        DRあり: DRに集約
   (5台で10本)                        (DROTHERどうしは隣接しない)

   R1 ─ R2                              R1   R2
   │ ╲ ╱ │                              │ ╲ ╱ │
   │  ╳  │      →                      [DR]─[BDR]
   │ ╱ ╲ │                              │ ╱ ╲ │
   R3 ─ R4                              R3   R4
   (R5省略)                             更新は 224.0.0.6(DR/BDR宛)へ

図4-3: マルチアクセス網でのDR/BDRによる隣接関係の集約。LSA交換の爆発を防ぐ仕組みだ。

DRの選出は、インタフェースごとに設定するプライオリティ値が最大のルータが勝ち、同点ならルータIDが最大のルータが勝つ。プライオリティ0のルータはDRに立候補しない。注意点として、DRの座は横取りされない(非プリエンプティブ)。後からプライオリティの高いルータが参加しても、現職のDRは交代しない。「設定上はこのルータがDRのはずなのに」という出題の引っかけどころだ。なお、ルータを1対1で直結するポイントツーポイント接続では集約の必要がないため、DR/BDRは選出されない。

試験ではこう出る「マルチアクセスネットワークでDRを選出する目的を述べよ」は記述の定番。解答の芯は「ルータ間の隣接関係とLSA交換をDRに集約し、経路情報交換の負荷を減らすため」。また「プライオリティが同じときは何で決まるか(→ルータID)」「DR障害時は誰が引き継ぐか(→BDR)」も午前IIで問われる。

4-6OSPF(2) — LSA・LSDBとコスト計算

隣接関係ができたルータは、LSA(Link State Advertisement)という単位で情報を交換する。LSAは「私はルータXで、こういうリンクをこういうコストで持っている」という自己申告書だ。各ルータのLSAはエリア内の全ルータにフラッディングされ、集まったLSAの束がLSDB(Link State Database)になる。同一エリア内の全ルータは、完全に同じLSDBを持つ——これがOSPFの大原則。そのうえで各ルータは、LSDBという地図の上で自分を起点にダイクストラ法(SPF計算)を実行し、全宛先への最短経路ツリーを作って、その結果をルーティングテーブルに反映する。

LSAには種別があり、試験で意識すべきは次の4つで足りる。

コストの計算 — 基準帯域÷インタフェース帯域

OSPFのメトリックはコストで、経路上の出力インタフェースのコストの合計が小さい経路が選ばれる。各インタフェースのコストは既定で基準帯域(100Mビット/秒)÷インタフェース帯域。つまり100Mビット/秒のリンクはコスト1、10Mビット/秒ならコスト10。計算結果が1未満になる高速リンクは切り上げて最小値の1になるため、既定のままでは1Gビット/秒も10Gビット/秒もコスト1で、速さの差を区別できない。実務では基準帯域を10Gビット/秒などに引き上げて対応するが、このとき全ルータで同じ基準帯域に揃える必要がある(揃えないとルータごとに地図の縮尺が変わり、経路計算が歪む)。この「1Gが区別できない」「全台で統一」の2点は午前II・午後の両方で狙われる。

例題 4-2図のOSPFネットワークで、R1からサーバセグメントN(R4に接続)への経路を考える。基準帯域は既定の100Mビット/秒とする。

(1) 経路1: 12、経路2: 4 (2) 経路2 (3) 経路1

経路2: R1→R3(1)+ R3→R5(1)+ R5→R4(1)+ R4→N(1)= 4

合格の視点コスト計算は「出力インタフェース側で数える」ことだけ間違えなければ単純な割り算と足し算だ。問題文の構成図に、各リンクのコストを書き込んでから経路を比較する——この一手間で計算ミスがほぼ消える。

4-7OSPF(3) — エリア設計と経路集約

OSPFは全ルータが同じ地図を持つ設計なので、ネットワークが大きくなると地図(LSDB)が肥大化し、SPF計算の負荷とフラッディングのトラフィックが増えていく。どこか1本のリンクがフラップ(断続)するだけで、全ルータが地図を書き直し計算をやり直す。この問題への答えがエリア分割だ。

OSPFではネットワークをエリアという単位に区切る。LSAのフラッディングとSPF計算はエリア内で閉じ、エリア内のルータは他エリアの詳細な地図を持たない(他エリアの情報はタイプ3LSAによる「経路の要約」として受け取るだけ)。これでLSDBのサイズも、障害時の再計算の波及範囲も、エリア内に限定できる。

エリア設計には守るべきルールがある。中心にバックボーンエリア(エリア0)を置き、他の全エリアはエリア0に直接接続する。エリア間のトラフィックは必ずエリア0を経由する、いわばハブ&スポーク構造だ。エリアの境界に立って複数エリアに足をかけるルータをABR(Area Border Router: エリア境界ルータ)と呼び、エリア間の経路情報(タイプ3LSA)はABRが生成する。また、OSPFの外の世界(BGP経路やスタティックルート)をOSPFに持ち込むルータはASBR(AS境界ルータ)と呼ばれ、タイプ5LSAを生成する。

              ┌───────── エリア0(バックボーン) ─────────┐
              │        [R10]──────[R11]                 │
              │        /              ╲                 │
              └─────[ABR1]──────────[ABR2]──────────────┘
                     /                  ╲
        ┌─ エリア1 ─┐          ┌─ エリア2 ─┐
        │ 10.1.0.0/22 │          │ 10.2.0.0/22 │
        │ [R1] [R2]  │          │ [R3] [R4]  │
        └────────────┘          └────────────┘
   ABR1はエリア1の経路をタイプ3LSAでエリア0へ広告
   (集約設定で 10.1.0.0/22 の1本にまとめられる)

ABRはエリア間で経路を渡すとき、経路集約(ルート集約)ができる。エリア1の中に10.1.0.0/24〜10.1.3.0/24の4サブネットがあるなら、ABRで10.1.0.0/22の1本に束ねてエリア0へ広告する。他エリアのルーティングテーブルは小さくなり、しかもエリア1内の個別サブネットがフラップしても、集約経路が変わらない限り他エリアには波及しない。集約はABR(とASBR)というエリアの境界でだけ実行できる——エリア内では全ルータが同じ地図を持つ必要があるからだ。集約アドレスの計算そのものは第3章のスーパーネット化(第3章)と同じ手順である。

試験ではこう出る午後の設計問題で「エリアを分割する利点を述べよ」ときたら、「LSDBの大きさとSPF再計算・フラッディングの範囲をエリア内に限定し、ルータの負荷と障害の波及を抑えるため」。「ABRで経路集約を行う利点」なら「他エリアへ広告する経路数を減らし、エリア内の経路変動を他エリアに波及させないため」。この2つの記述はほぼ定型文として書けるようにしておこう。集約範囲を答えさせる計算(例: 4つの/24を/22に)もセットで出る。

4-8BGP — AS間ルーティングとパス属性による経路制御

OSPFが「組織の中」の最短経路を自動計算するプロトコルだとすれば、BGP(Border Gateway Protocol)は「組織と組織の間」で経路を交換するプロトコルだ。インターネットは、ISPや大企業といったAS(自律システム)——ひとつの管理ポリシーで運用されるネットワークのかたまり——が相互接続してできている。各ASにはAS番号が割り当てられ、AS間の経路交換はBGP(現行版はBGP-4)が一手に担う。インターネットの経路表そのものを支えているプロトコルであり、NW試験でもインターネット接続・拠点間WAN・データセンタ設計の文脈で登場する。

BGPの接続形態は2種類ある。異なるAS間で張るセッションがeBGP、同じAS内のルータどうしで張るのがiBGPだ。iBGPは「eBGPで学んだ経路をAS内の他の出入口ルータへ運ぶ」役割を持つ。BGPのセッションはTCP(ポート179)上に張られる点も特徴で、OSPFのようなマルチキャストによる自動発見はせず、対向を明示的に設定して1対1で接続する。信頼性はTCPに任せ、BGP自身は経路交換のポリシー制御に専念する設計だ。

BGPはディスタンスベクタの親戚だが、距離の代わりに経路が通ってきたASの並び(AS_PATH)を運ぶためパスベクタ型と呼ばれる。受け取った経路のAS_PATHに自分のAS番号が既に含まれていたらその経路を破棄する——これがBGPのループ防止であり、AS_PATHが「メトリック」と「ループ検知」を兼ねる。

パス属性 — BGPは「最短」ではなく「ポリシー」で選ぶ

BGPの経路にはパス属性という付加情報が付いており、複数の経路候補から1つを選ぶ(ベストパス選択)ときの判断材料になる。試験で使い分けを問われるのは次の3つだ。

(3) AS_PATH長の比較は各ASのベストパス選択の一要素にすぎず、途中のASが自身のポリシー(LOCAL_PREFなど)でISP-B経由を優先すれば、プリペンドは覆されるから。着信制御は「相手への働きかけ」であり、最終決定権は各ASにある——この非対称性はBGP理解の核心で、記述問題の題材になる。

4-9経路再配布の注意点と、ホスト側の冗長化への橋渡し

現実のネットワークでは、複数のルーティングプロトコルが同居することがある。合併した会社のOSPF網とRIP網をつなぐ、BGPで受けたWANの経路を社内OSPFに流す、スタティックルートをOSPFに広告する——こうした場面で、あるプロトコルが持つ経路情報を別のプロトコルに注入する操作を経路再配布(リディストリビューション)と呼ぶ。便利だが、試験ではもっぱら「事故のもと」として出題される。押さえるべき注意点は2つ。

  • メトリックは変換できない — OSPFのコストとRIPのホップ数とBGPの属性は物差しが違うため、再配布時に元のメトリックは意味を失う。再配布する側で注入先プロトコルの流儀のメトリック(シードメトリック)を明示的に与える必要がある。与え方が不適切だと、意図しない経路が優先されたり、経路が到達不能扱いになったりする。
  • ループと経路の逆流 — 2台以上のルータで双方向に再配布すると、プロトコルAから注入した経路がプロトコルBを一周してAに「新しい経路」として戻ってくることがある。宛先は同じなのに情報源が変わったことでAD値の序列が変わり、本来の経路より再配布経由の遠回り経路が採用される——という形で、経路ループや最適でない経路選択が発生する。対策は、再配布を一方向に限定する、再配布した経路にタグを付けて再注入をフィルタで拒否する、AD値を調整して元経路を優先させるなど。「どこで何が逆流するか」を構成図上で追わせる午後問題が実際に出る。
合格の視点再配布の問題は、パケットではなく経路情報の流れを図に矢印で書き込むと一気に見通せる。「経路情報がどのプロトコルからどのプロトコルへ、どのルータで乗り換えるか」を追う——転送の話(データプレーン)と経路交換の話(コントロールプレーン)を分けて考えるのは、NW午後問題全般に効く思考法だ。

最後に、この章で学んだルーティングの世界の「外側」に触れておく。ルーティングプロトコルはルータどうしの経路切替を自動化するが、エンドホスト(PCやサーバ)はルーティングプロトコルに参加していない。ホストが知っているのは自分のデフォルトゲートウェイのIPアドレスだけだ。つまり、デフォルトゲートウェイのルータが故障すると、その先のネットワークがどれだけ冗長でも、ホストは一歩も外に出られない。この「最初の1ホップ」を守る仕組みがVRRP(Virtual Router Redundancy Protocol)で、複数のルータを1つの仮想ルータに見せてゲートウェイを冗長化する。ルーティングプロトコルとVRRPは守備範囲が違う相棒であり、両方を組み合わせて初めて端から端まで冗長になる。詳しくは第9章(冗長化・負荷分散・運用管理)で扱う。

試験ではこう出る「ルータ更改」「ネットワーク改善」系の午後問題では、OSPFのエリア・コスト設計とVRRP・フローティングスタティックが1つの構成図の中に同居して出題される。それぞれの技術がどの障害パターンを救うのか(回線断→ルーティング/フローティングスタティック、GW故障→VRRP)を対応付けて説明できるようにしておくと、設問の意図が読みやすくなる。

正解: ウ — リンクステート型(OSPF)は全ルータが同一のLSDB(地図)を持ち、各自でSPF計算を行う。アとイはディスタンスベクタ型(RIP)の説明。エは前半(ホップ数)がRIPの特徴なのに後半(速度差を反映)がホップ数では実現できない内容で、文として矛盾している——こういう「前半と後半のねじれ」は誤答肢の常套手段だ。

Q5. OSPFのマルチアクセスネットワークでDR/BDRを選出する目的として、最も適切なものはどれか。
  • ア セグメント内のルータ間で隣接関係とLSA交換を代表ルータに集約し、経路情報交換の負荷を減らすため
  • イ セグメント内のパケット転送を代表ルータに集中させ、転送経路を一意にするため
  • ウ エリア間の経路情報を集約して広告するルータを決めるため
  • エ Helloパケットの送信をDRだけに限定し、帯域を節約するため

正解: ア — DR/BDRは「経路情報交換(コントロールプレーン)の集約」のための仕組み。イはデータの転送(データプレーン)の話で、DRを経由してパケットが転送されるわけではない——ここが最も紛らわしい引っかけ。ウはABRの役割。エは誤りで、HelloはDROTHERを含む全OSPFルータが送信し続ける(ネイバーの死活監視に必要)。

Q6. OSPFの基準帯域が既定(100Mビット/秒)のとき、既定設定のままでは経路選択上区別できない組合せはどれか。
  • ア 10Mビット/秒と100Mビット/秒のリンク
  • イ 100Mビット/秒と1Gビット/秒のリンク
  • ウ 1Gビット/秒と10Gビット/秒のリンク
  • エ 10Mビット/秒と1Gビット/秒のリンク

正解: ウ — コスト=100M÷帯域で、1未満は1に切り上げ。1Gも10Gも計算上1未満となりコスト1で同値、つまり区別できない。イは100Mがコスト1、1Gも切り上げでコスト1……と思いきや100M÷100M=ちょうど1なので両方コスト1で区別できない?——いや、100Mはコスト1、1Gも最小値1で確かに同値になる。ただし設問は「既定のままでは区別できない代表例」を問うており、試験ではウ(ギガ超級どうし)が定番。イも同コストになる点に気づけた人は、基準帯域引き上げの必要性をより深く理解できている。実務・試験とも「高速リンクが混在したら基準帯域を全ルータで統一して引き上げる」が結論だ。

Q7. OSPFのエリア設計に関する記述のうち、適切なものはどれか。
  • ア 全てのエリアは互いに直接接続されていなければならない
  • イ エリア間の経路情報はABRが生成し、エリア0を介して他エリアへ伝えられる
  • ウ 経路集約はエリア内の任意のルータで実行できる
  • エ エリアを分割すると、全ルータが保持するLSDBは全エリアの詳細情報を含むようになる

正解: イ — エリア間経路(タイプ3LSA)はABRが生成し、エリア間トラフィックはバックボーン(エリア0)経由が原則。アは誤り——必須なのは「各エリアとエリア0の接続」であり、エリアどうしの全結合ではない。ウは誤りで、集約はエリアの境界(ABR/ASBR)でのみ可能。エは逆で、エリア分割の目的はLSDBを自エリアの詳細+他エリアの要約に絞ることだ。

Q8. BGPに関する記述のうち、適切なものはどれか。
  • ア eBGPは同一AS内のルータ間、iBGPは異なるAS間で経路を交換する
  • イ セッションはUDP上に張られ、経路情報はマルチキャストで一斉配布される
  • ウ 受信した経路のAS_PATHに自ASの番号が含まれていれば、その経路を破棄してループを防ぐ
  • エ メトリックにリンクの帯域幅を用い、最も高速な経路を自動選択する

正解: ウ — AS_PATHによるループ検知はBGP(パスベクタ型)の中核機構。アはeBGPとiBGPの説明が逆。イは誤りで、BGPセッションはTCPポート179の1対1接続であり、マルチキャストによる自動発見は行わない。エはOSPF的な発想の誤答で、BGPは帯域ではなくポリシー(パス属性)で経路を選ぶ。

Q9. 2つのISPに接続したマルチホーム構成の企業ASが、「自社から外部へ出ていくトラフィックの出口を主回線側に寄せる」ために自AS内で設定するBGPパス属性として、最も適切なものはどれか。
  • ア 主回線経由で受信した経路のLOCAL_PREFを大きくする
  • イ 予備回線へ広告する経路にAS_PATHプリペンドを行う
  • ウ 主回線経由で受信した経路のMEDを小さくする
  • エ 予備回線経由で受信した経路のネクストホップを書き換える

正解: ア — 出ていくトラフィックの出口選択は自AS内のベストパス選択の問題なので、iBGPで共有されるLOCAL_PREFで制御する。イは「入ってくる」トラフィックの制御手段であり方向が逆——本問最大の引っかけ。ウのMEDは自分が受信した経路に付けても意味がなく、隣接ASへ「入口はこちら」と伝えるための属性。出口=LOCAL_PREF、入口=プリペンド/MEDという方向の対応を確実に。

Q10. 2台のルータでOSPFとRIPの双方向再配布を行ったところ、一部の宛先への通信が最適でない経路を通るようになった。原因として最も適切なものはどれか。
  • ア OSPFのコストがRIPのホップ数に自動変換され、値が大きくなりすぎたから
  • イ 一方のプロトコルへ再配布された経路が他方を経由して元のプロトコルに逆流し、本来の経路より優先されたから
  • ウ 再配布によってLSAのフラッディング範囲がRIP側にも拡大し、収束が遅れたから
  • エ RIPのホップ数上限が16に制限され、OSPF由来の経路が全て到達不能になったから

正解: イ — 複数地点での双方向再配布では、経路情報が他プロトコルを一周して逆流し、AD値やメトリックの評価が変わって遠回り経路が採用されることがある。対策はタグによるフィルタや一方向化。アは誤り——メトリックは自動変換されない(物差しが違う)からこそシードメトリックを手動で与える。ウはLSAがRIP側に流れる事実がなく誤り。エは「全て到達不能」なら通信不可のはずで、「最適でない経路を通る」という症状と合わない。
